A) POWER ARCHITECTURE (UPDATED)
================================
Power assumptions:
- Battery:    3 x AAA NiMH rechargeable (about 4.2V fresh to about 3.0V)
- U1:         MCP1700-3302 (always-on XBee rail)
- U2:         XC6220A331MR-G (gated sensor rail using CE/EN)
- XBee rail:  VCC (pin 1) = 3.3V_A always-on

Always-on rail:
- 3xAAA(+) -> U1 VIN
- U1 VOUT  -> 3.3V_A -> XBee pin1 (VCC), pin14 (VREF), RPU source

Gated sensor rail:
- 3xAAA(+) -> U2 VIN
- U2 VOUT  -> 3.3V_S -> PMOS source (strong pull-up supply)
- XPIN13   -> R4 1k -> U2 CE/EN
- U2 CE/EN -> R3 100k -> GND (default OFF)

State behavior:
- XPIN13 HIGH (awake) -> U2 ON  -> 3.3V_S active
- XPIN13 LOW  (sleep) -> U2 OFF -> 3.3V_S off


B) 1-WIRE BUS + STRONG PULL-UP
===============================
DS18B20 parasite correction remains mandatory:
- DS1 pin1 (GND) -> GND
- DS1 pin2 (DQ)  -> DQ net
- DS1 pin3 (VDD) -> GND

Topology:
- RPU 4.7k from 3.3V_S to DQ
- PMOS source to 3.3V_S
- PMOS drain to DQ
- PMOS gate to XPIN12 via RG (100 to 180 ohm)
- PMOS gate bias resistor per board convention (keep existing validated topology)

Control polarity:
- XPIN12 LOW  -> PMOS ON  -> strong pull-up active during Convert T
- XPIN12 HIGH -> PMOS OFF -> weak pull-up only


C) FIRMWARE CONTROL (XBee API / AT)
===================================
Strong pull-up sequence:
1) Assert XPIN12 LOW -> PMOS ON
2) Issue Convert T (0x44)
3) Wait 750 ms
4) Deassert XPIN12 HIGH -> PMOS OFF
5) Read scratchpad (0xBE)

Sensor-rail gating:
- XPIN13 is managed by XBee sleep state.
- Ensure CE/EN sees valid HIGH when awake and LOW when asleep.

Safe defaults:
- XPIN12 default HIGH at boot (PMOS OFF)
- CE/EN pull-down (R3) keeps U2 OFF if XPIN13 floats during startup
