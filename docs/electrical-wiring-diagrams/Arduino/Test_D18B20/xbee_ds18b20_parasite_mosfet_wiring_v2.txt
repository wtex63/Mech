A) NEW SCHEMATIC WIRING (for PCB revision)
==========================================
Power assumptions:
- Battery:    2 x 1.5V rechargeable lithium cells
- Regulator:  XC9140 boost converter
- XBee rail:  VCC (XBee pin 1) = 3.3V nominal

DS18B20 component DS1 corrections:
- DS1 pin 1 (GND) → GND           [XBee pin 10]
- DS1 pin 2 (DQ)  → DQ net        (1-Wire bus node)
- DS1 pin 3 (VDD) → GND           ← tie to GND for parasite power; do NOT leave NC

1-Wire bus and strong pull-up topology:

   VCC (pin 1) ─────┬──────────────────────────────────────────┐
                    │                                           │
                 RPU 4.7k                                  AO3401A
                    │                                      (P-MOSFET)
                    └──── DQ ─────────────────────────── DRAIN
                                                              SOURCE ──── VCC (pin 1)
                                                              GATE
                                                               │
                                         ┌──────── RG 100 Ω ──┘
                                         │
                       DIO11/PWM1/PTA2/SDA ────┤     ← XBee XPIN7, old R2/BC847 base net
                                                │
                                                └──────── RGP 100 kΩ ──── VCC (pin 1)

                    DQ ─────────────────────────── DIO12/PTB4/MISO1  ← XBee XPIN4

Resistor values:
- RPU  = 4.7k    (weak pull-up, DQ to VCC)
- RG   = 100 Ω   (gate series, suppresses oscillation; optional but recommended)
- RGP  = 100 kΩ  (gate pull-up holds MOSFET OFF at idle/startup)

Note: Original schematic R2 (4.7k near T1/BC847) → repurpose as RPU.
      R2 trace to BC847 base is cut; new RG/RGP network bodged at MOSFET.

Logic behavior (P-MOSFET high-side switch, active-LOW):
- DIO11/PWM1/PTA2/SDA (XPIN7) = LOW  → MOSFET ON  → strong pull-up active (Convert T window)
- DIO11/PWM1/PTA2/SDA (XPIN7) = HIGH → MOSFET OFF → only RPU 4.7k weak pull-up active

Polarity change vs. BC847:
- Old: XBee DIO HIGH → BC847 ON → strong pull-up
- New: XBee DIO LOW  → AO3401A ON → strong pull-up
→ FIRMWARE MUST INVERT the strong pull-up GPIO assertion.

Why this fixes BC847 Vbe issue:
- BC847 emitter follower: DQ = VCC − 0.7V = 3.3V − 0.7V = 2.6V during Convert T
  DS18B20 minimum parasite Vdd = 3.0V → marginal / out of spec
- AO3401A passes near-full VCC to DQ (Vds(on) << 0.1V)
  DQ ≈ VCC = 3.3V during Convert T → fully within spec


B) REWORK PLAN FOR ALREADY-MANUFACTURED PCBS
=============================================
Goal: Keep existing board and bodge-replace T1 (BC847) with AO3401A.

Step 1 — Pre-checks (DMM, power OFF)
-------------------------------------
a) Confirm DS1 pin 3 (VDD) is tied to GND.
   Net name in schematic: GND  (XBee pin 10)
   If currently NC, add bodge wire DS1 pin 3 → GND.

b) Identify these physical pads on PCB:
   - VCC        (XBee pin 1 trace / XC9140 VOUT)
   - GND        (XBee pin 10 / battery −)
   - DQ         (DS1 pin 2 trace; also R1 top end if present)
   - XPIN7      (DIO11/PWM1/PTA2/SDA; pad connected to R2 / old BC847 base)

c) Record DMM resistance R2:
   - If R2 ≈ 100 Ω → may reuse in-circuit as RG (check current headroom)
   - If R2 ≈ 4.7 kΩ → use as RPU (disconnect from BC847, route to DQ instead)
   - If unsure → fully lift R2 and fit new RG (100 Ω) and RGP (100 kΩ) fresh

Step 2 — Remove or isolate BC847 (T1)
---------------------------------------
Remove T1 from pads, or:
  - Lift emitter lead (the DQ-connected leg) with solder wick + hot air
  - Verify DMM: no continuity VCC → DQ through T1 after removal

Step 3 — Dead-bug AO3401A wiring (SOT-23)
-------------------------------------------
Verify pin-1 (Source/Drain) from your vendor datasheet before soldering.
Common AO3401A SOT-23 pinout: pin1=Gate, pin2=Source, pin3=Drain.

  AO3401A SOURCE → VCC          (enamel bodge to VCC plane/pad)
  AO3401A DRAIN  → DQ           (enamel bodge to DQ trace)
  AO3401A GATE   ──── RG 100 Ω ────── XPIN7 pad      (DIO11/PWM1/PTA2/SDA; old R2 / BC847 base)
  AO3401A GATE   ──── RGP 100 kΩ ──── VCC            (gate pull-up)

If old R2 is 100 Ω and still on board:
  Keep R2 in place; connect old BC847-base pad → new GATE via R2 directly.
  Add RGP 100 kΩ from GATE node → VCC.

Step 4 — Verify (DMM, power OFF)
----------------------------------
1) DQ → VCC via 4.7k RPU: YES
2) GATE → VCC via 100k RGP: YES
3) DS1 pin3 → GND: YES
4) No hard short VCC ↔ GND: YES

Step 5 — Scope check (power ON)
---------------------------------
1) Idle DQ ≈ VCC (3.3V)
2) Assert DIO11/PWM1/PTA2/SDA (XPIN7) LOW; scope DQ → rises to ≈ VCC (not VCC−0.7V)
3) Deassert HIGH; DQ held by 4.7k RPU ≈ VCC
4) Full Convert T: DQ stays ≥ 3.0V throughout 750ms window


C) FIRMWARE CONTROL (XBee API / AT commands)
=============================================
Confirmed pin mapping for this design:
- DQ data line: DIO12/PTB4/MISO1 (XPIN4)
- MOSFET gate control: DIO11/PWM1/PTA2/SDA (XPIN7)

AT command usage:
- DIO11/PWM1 is controlled by AT command P1 on Zigbee S2B firmware.
- Verify once in XCTU for your exact firmware build before final deployment.

Strong pull-up assertion sequence per Convert T:
1) ATP1=4 (set output LOW  → MOSFET ON  → strong pull-up)   ← active-LOW
2) Issue Convert T (0x44) on DQ
3) Wait 750ms (12-bit conversion)
4) ATP1=5 (set output HIGH → MOSFET OFF → weak pull-up only)
5) Issue Read Scratchpad (0xBE) on DQ

Safe default at XBee boot:
- Set P1 direction = OUTPUT, default = HIGH
  → MOSFET stays OFF until firmware explicitly asserts LOW
  → no accidental hard pull VCC → DQ at startup

  "DS1 pin 3 VDD → GND  (parasite power mode)"
  "Q1 replaces T1 BC847: no Vbe drop, DQ ≈ VCC during Convert T"

F) FINAL PCB FRONT-END DRAFT (XPIN4/XPIN7/XPIN11/XPIN15/XPIN20)
=================================================================
Design intent:
- Zigbee enabled final product
- DS18B20 in parasite mode
- Tilt switch input with RC + ESD hardening
- Association LED on XPIN15
- Commissioning button on XPIN20

Pin allocation (final):
- XPIN4  (DIO12/PTB4/MISO1): DS18B20 DQ bus
- XPIN7  (DIO11/PWM1/PTA2/SDA): Strong pull-up MOSFET gate control (active-LOW)
- XPIN11 (DIO4/PTB3/MOSI1): Tilt switch digital input
- XPIN15 (ASSOC/DIO5/PTD4): Association LED (keep Digi special function)
- XPIN20 (DIO0/ADC0/PTA0): Commissioning button (keep Digi special function)

1) DS18B20 (XPIN4) front-end
----------------------------
Keep existing parasite topology:
- RPU_DQ = 4.7k from DQ to VCC
- Q1 = AO3401A (or GSF3407) strong pull-up path
  Source -> VCC
  Drain  -> DQ
  Gate   -> XPIN7 through RG = 100 ohm
  Gate   -> VCC through RGP = 100k

Optional DQ ESD when sensor lead exits enclosure:
- Add very low-capacitance TVS from DQ to GND near connector
- Target capacitance <= 5 pF to avoid 1-Wire edge distortion
- If cable is long/noisy, optional series resistor 22 to 47 ohm in DQ at connector side

Do NOT place 100 nF directly on DQ.

2) Tilt switch input (XPIN11) front-end
---------------------------------------
Recommended electrical (low-power, switch-to-VCC):
- R_TILT_PD = 1M from XPIN11 node to GND
- C_TILT    = 100 nF from XPIN11 node to GND
- Tilt switch contact from XPIN11 node to VCC (active-HIGH)

Optional cable/input protection:
- TVS_TILT: low-cap TVS from tilt signal to GND, placed at connector entry
- R_TILT_SER: 100 to 330 ohm series from connector signal to XPIN11 node

Preferred chain for external wiring:
Connector_TILT -> TVS to GND -> R_TILT_SER -> XPIN11_RC_NODE -> XPIN11

Firmware:
- Configure XPIN11 as digital input (external pull-down used)
- Add debounce 20 to 50 ms in software

3) LED and commissioning pins
-----------------------------
XPIN15 (Association LED):
- Keep under Digi special/association handling
- Typical LED path: XPIN15 -> R_LED (1k to 2.2k) -> LED -> GND
- Verify polarity against your existing board convention

XPIN20 (Commissioning button):
- Keep under Digi commissioning function
- Button to GND with pull-up enabled in config
- If external button wiring is long, use same ESD pattern as tilt input

4) Single-sided board layout rules (ground plane present)
---------------------------------------------------------
Placement priority:
1) Place TVS parts at external connector entry points first
2) Place RC parts adjacent to the MCU/XBee input node second
3) Keep XPIN4 DQ path short and away from LED and switch traces
4) Keep Q1 + RG + RGP cluster tight around DQ/VCC/XPIN7

Grounding:
- Use short, wide return from each TVS to ground plane
- Keep RC capacitor ground stub short (<5 mm preferred)
- Avoid sharing high-current LED return path with DQ/tilt signal returns

Routing notes:
- Do not run DQ parallel to LED drive trace for long distance
- Avoid stubs on DQ; use point-to-point style where practical
- Keep connector-to-TVS segment very short
