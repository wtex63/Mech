# Contributing

## Branching

- Use short topic branches from main.
- Recommended prefixes: feat/, fix/, docs/, chore/.

## Pull Requests

- Keep changes focused and reviewable.
- Include a short summary of design impact.
- Attach images or screenshots for visual CAD changes when useful.

## File Organization

- Design standards go in docs/.
- Fusion scripts belong in a360/scripts/.
- Post files belong in a360/post-processors/.
- Checklists and templates belong in templates/.

## Commit Messages

Use imperative style:

- add enclosure wall thickness rule
- update naming convention for fixtures
- add fusion bom export script scaffold
